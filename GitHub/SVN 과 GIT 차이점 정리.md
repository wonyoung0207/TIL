# SVN vs Git 차이점 핵심 정리

---

>

## 1. 버전 관리 구조

- **SVN**: 중앙 집중형 (서버에만 전체 히스토리 존재)
- **Git**: 분산형 (모든 로컬에 전체 히스토리 존재)

------

## 2. 서버 의존성

- **SVN**: 서버 없으면 작업 거의 불가
- **Git**: 서버 없이도 로컬 작업 가능

------

## 3. 커밋 방식

- **SVN**: 커밋 = 바로 서버 반영
- **Git**: 로컬 커밋 → 필요 시 서버 푸시

------

## 4. 브랜치 개념

- **SVN**: 디렉토리 복사 (무겁고 제한적)
- **Git**: 포인터 기반 (가볍고 자유로움)

------

## 5. 오프라인 작업

- **SVN**: 거의 불가능
- **Git**: 완전 가능

------

## 6. 성능

- **SVN**: 서버 통신 많아 느림
- **Git**: 로컬 기반으로 매우 빠름

------

## 7. 협업 방식

- **SVN**: 중앙 기준, 충돌은 커밋 시점에 발생
- **Git**: 로컬 병합 후 공유, 충돌 제어 용이

------

## 8. 현대 개발 적합성

- **SVN**: 레거시 / 중앙 통제 환경
- **Git**: 현대 표준 / CI·CD·대규모 협업

------

## 한 줄 요약

> **SVN = 중앙 서버 중심 관리**
>  **Git = 개발자 중심 분산 관리**
