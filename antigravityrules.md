# Antigravity Rules: TIL Project Guidelines

이 파일은 **TIL (Today I Learned)** 레포지토리의 프로젝트 구조, 관리 규칙, 그리고 AI 어시스턴트(Antigravity)와의 협업 가이드를 정의합니다.

## 1. 프로젝트 목적 (Purpose)
이 프로젝트는 다음과 같은 두 가지 핵심 목적을 가집니다.
1.  **지식 공간 (Knowledge Space)**: 개발 공부 내용, 기술 개념, 문제 해결 과정을 기록하는 'Second Brain'입니다.
2.  **포트폴리오 (Portfolio)**: 학습한 내용과 진행한 프로젝트를 체계적으로 보여주는 커리어 관리 공간입니다.

## 2. 디렉토리 구조 (Directory Structure)
루트 디렉토리는 대분류(Category)를 의미하며, 모든 새로운 지식과 프로젝트는 적절한 카테고리 안에 위치해야 합니다.

### 구조 원칙
*   **루트(Root)**: `Java`, `Spring`, `React` 등 기술 스택이나 주제별 대분류 폴더.
*   **서브폴더(Sub-folder)**:
    *   **개념 정리**: 특정 주제에 대한 마크다운 파일들.
    *   **프로젝트**: 소스 코드가 포함된 실습 또는 토이 프로젝트. 각 프로젝트 폴더는 반드시 해당 프로젝트를 설명하는 `README.md`를 포함해야 합니다.

## 3. 파일 및 명명 규칙 (Conventions)
*   **파일 형식**: 문서는 기본적으로 **Markdown (`.md`)** 형식을 사용합니다.
*   **언어**: 주 사용 언어는 **한국어(Korean)**입니다.
*   **이름 짓기**:
    *   내용을 명확하게 파악할 수 있는 직관적인 이름을 사용합니다.
    *   예: `HashMap과 LinkedHashMap의 차이점.md`, `Spring Security 설정가이드.md`
    *   가독성을 위해 **띄어쓰기(Space)**를 적극적으로 사용합니다. `_`(Underscore) 대신 공백을 사용하세요.

## 4. 관리 워크플로우 (Workflow)
새로운 지식이나 프로젝트를 추가할 때는 다음 단계를 따릅니다.

1.  **위치 선정**: 주제에 맞는 폴더를 찾거나, 대분류가 없다면 새로 생성합니다.
2.  **문서 작성**: 배운 내용, 코드 스니펫, 참고 링크 등을 포함하여 마크다운 문서를 작성합니다.
3.  **인덱스 업데이트**:
    *   새로운 대분류나 중요한 프로젝트가 추가된 경우, 루트 디렉토리의 **`README.md`**에 링크를 추가하여 목차를 최신화합니다.
    *   `README.md`는 포트폴리오의 대문 역할을 하므로 항상 깔끔하게 유지합니다.

## 5. Antigravity 가이드 (Agent Protocol)
AI 어시스턴트(Antigravity)는 이 레포지토리를 작업할 때 다음 원칙을 준수해야 합니다.

*   **구조 존중**: 사용자의 기존 폴더 구조를 무단으로 변경하지 않으며, 새로운 파일은 가장 적합한 기존 폴더에 배치합니다.
*   **문맥 파악**: 작업 전에 항상 `antigravityrules.md`를 참고하여 프로젝트의 성격을 상기합니다.
*   **제안**: 포트폴리오로서의 가치를 높일 수 있는 구조 개선이나 문서화 방안이 있다면 사용자에게 적극적으로 제안합니다.
