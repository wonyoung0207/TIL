# JIT 개념정리

---

>

## JIT란?

> 프로그램을 **실행하는 순간(런타임)** 에 기계어로 변환하는 컴파일 방식

즉,

- 미리 전부 기계어로 만드는 것도 아니고 (AOT 아님)
- 한 줄씩 해석만 하는 것도 아님 (인터프리터 아님)
- **실행 중에 필요한 부분만 기계어로 바꿔서 최적화**하는 방식
  - JAVA는 빌드시 기계어가 아닌 바이트 코드(.class) 로 변경됨 

------

## 사용 이유 

- 플랫폼 독립성과 실행 속도를 **둘 다 잡기 위해서**
- 예: Java

```
.java → 빌드 → .class (바이트코드)
```

- 이 바이트코드는 CPU가 직접 실행 못함.

- 실행 시:

```
.class → JVM
            ↓
         JIT 컴파일
            ↓
         기계어 실행
```

------

## 동작 원리

1. 프로그램 시작
2. 바이트코드 실행
3. 자주 실행되는 코드(Hot Code) 발견
4. 해당 부분을 기계어로 변환
5. 이후부터는 빠르게 실행

- 많이 쓰는 부분만 최적화함
- 이를 **Hot Spot 최적화**라고 함.

------

## 실행 방식 비교

| 구분          | 인터프리터 | AOT     | JIT       |
| ------------- | ---------- | ------- | --------- |
| 컴파일 시점   | 실행 중    | 실행 전 | 실행 중   |
| 속도          | 느림       | 빠름    | 매우 빠름 |
| 플랫폼 독립성 | 높음       | 낮음    | 높음      |
| 동적 최적화   | 불가       | 제한적  | 가능      |

------

## 장점 및 단점

1. 장점
   - 실행 중 최적화 가능
   - 플랫폼 독립성 유지
   - 반복 코드에서 매우 빠름
2. 단점
   - 초기 실행이 약간 느림
   - 메모리 사용 증가
   - 구현 복잡

------

## JIT 사용 예

- **Java (JVM)**
- **C# (.NET CLR)**
- **JavaScript (V8 엔진)**

------

## 한 줄 요약

> JIT는 프로그램을 실행하면서 자주 쓰이는 코드를 기계어로 변환해 점점 빠르게 만드는 컴파일 기술이다.
