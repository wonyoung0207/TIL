# 3D Model 변형값 정규화

---

>

## 역할

- 블렌더에서 `Ctrl + A`는 **'Apply(적용)'** 메뉴를 호출하는 단축키입니다. 3D 공간에서 오브젝트의 변형(Transform) 값을 **데이터 자체에 완전히 고정**시키는 역할을 한다. 
- **"현재까지의 변수값들을 하드코딩된 상수로 변환하는 과정"**이라고 볼 수 있다. 

------

## `Ctrl + A` 메뉴의 주요 항목들

### 1. Location (위치)

- **기능:** 오브젝트의 현재 위치를 `(0, 0, 0)`으로 초기화한다. 
- **결과:** 오브젝트는 그 자리에 가만히 있지만, '오브젝트의 중심점(Origin)'은 3D 월드의 원점으로 이동
- **용도:** 여러 오브젝트의 원점을 한곳으로 일치시켜야 할 때 사용한다. 

### 2. Rotation (회전)

- **기능:** 현재 기울어진 상태를 `(0, 0, 0)`으로 초기화합니다.
- **결과:** 화면상에서는 기울어져 있지만, 블렌더 데이터상으로는 "이게 똑바로 서 있는 상태야"라고 인식하게 된다. 
- **용도:** 리깅 시 관절의 회전축을 정렬할 때 필수이다.

### 3. Scale (크기)

- **기능:** 현재 크기를 `(1, 1, 1)`로 초기화합니다.
- **결과:** 크기를 5배 키웠더라도 `Apply Scale`을 누르면 다시 1배가 된다. .
- **용도:** 베벨(Bevel)이 찌그러지거나, 텍스처가 늘어나는 현상을 방지할 때 사용한다. 

### 4. All Transforms (모든 변형 적용)

- **기능:** 위 3가지(위치, 회전, 크기)를 한꺼번에 적용한다. 
- **용도:** 모델링을 마친 후 리깅이나 익스포트 전, 데이터를 '깨끗하게' 정리할 때 가장 많이 사용한다. 

------

### Apply의 데이터 구조

> ```
> ┌─────────────────────────────────────────────────────────────────────────┐
> │ [ 📋 블렌더의 데이터 레이어 이해 ]                                         │
> │                                                                         │
> │  1. 오브젝트 데이터 (Object Transform) - [변수]                           │
> │     "이 상자를 2배 키우고 45도 돌려라." (언제든 수정 가능)                    │
> │                                                                         │
> │  2. 메쉬 데이터 (Mesh Geometry) - [원본]                                 │
> │     상자를 구성하는 점(Vertex)들의 좌표값.                                  │
> │                                                                         │
> │  ★ Ctrl + A 를 누르면? ★                                                 │
> │     오브젝트의 변수값(2배, 45도)을 계산해서                                  │
> │     메쉬 데이터의 점 좌표에 직접 더하고 곱해버림.                            │
> │     그 후 변수는 다시 1배, 0도로 리셋됨!                                    │
> └─────────────────────────────────────────────────────────────────────────┘
> ```

------

## 하는이유 

- 만약 `Apply`를 하지 않은 채로 **WebGL(Three.js)** 등으로 모델을 가져가면 다음과 같은 '버그'가 생긴다. 

1. **Scale 오류:** 웹 엔진에서 `scale.set(1, 1, 1)`을 주었는데, 블렌더에서 실제 메쉬가 작고 스케일 변수만 컸다면 엔진에서는 아주 작게나옴 
2. **Rotation 오류:** 로봇 팔 관절을 돌리는데, 로컬 축이 엉뚱하게 뒤틀려 있어 `rotation.x`를 바꿨는데 팔이 옆으로 꺾이는 현상이 발생

- 한마디로 `Ctrl + A`는 **"내가 눈으로 보고 있는 이 상태를 진짜 데이터로 확정 짓겠다"**는 명령이다.
