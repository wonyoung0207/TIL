# Rigging 시 Rotation and Scale 적용하는 이유

---

>

## 사용 명령어

1. `A `
   1. Riggig 할 Model 전체 선택
2. `Ctrl + A` 
   1. Apply 설정창 
3. Rotation & Scale 선택

## 필요한 이유 

- 리깅(Rigging)이나 애니메이션 작업을 시작하기 전에 `Ctrl + A`를 눌러 **Rotation & Scale**을 적용(Apply)하는 것은 
  - 모델의 **'근본(기본값)'을 바로잡는 작업**이다.
- 이 과정을 생략하면 나중에 로봇 팔이 엿가락처럼 휘거나, 엉뚱한 방향으로 돌아가는 대참사가 발생할 수 있기 때문이다. 

------

### 1. "현재 상태를 0(회전)과 1(스케일)로 초기화"

- 블렌더에서 오브젝트를 크기 조절하거나 회전하면, 블렌더는 내부적으로 두 가지 값을 가진다. 
  - **Mesh Data:** 실제 점(Vertex)들의 좌표
  - **Object Transform:** 이 메쉬를 얼마나 키웠고 돌렸는지에 대한 변수 값
- `Apply`를 하지 않으면 메쉬 데이터는 그대로인데 겉모습만 뻥튀기된 상태이다. 
  - `Apply`를 누르는 순간, 현재의 커진 크기를 **"이게 원래 1:1 사이즈야"**라고 데이터 자체에 설정하는것이다. 

### 2. 리깅(Rigging) 시 계산 오류 방지

- 로봇 팔 리깅의 핵심인 **IK(Inverse Kinematics)**는 수학적 연산을 통해 각 관절의 각도를 계산한다. 
  - 만약 스케일이 `1.5`로 되어 있다면, 뼈(Bone)가 움직일 때 메쉬가 1.5배 더 민감하게 반응하거나 찌그러지는 현상이 발생하게 된다. 
  - 회전값이 `0`이 아니라면, 뼈의 '앞쪽(Forward)'과 메쉬의 '앞쪽'이 일치하지 않아 로봇 팔이 관절 꺾이듯 이상하게 움직이게 된다. 

------

### 요약 

- 웹 환경에서 `Object.scale.set(1, 1, 1)`인데 실제로는 엄청 거대하게 보이는 문제를 방지하려면, 반드시 블렌더에서 **Apply**를 완료한 상태로 내보내야 한다. 

> ```
> ┌─────────────────────────────────────────────────────────────────────────┐
> │ [ Apply(Ctrl+A)를 안 하면 생기는 일 ]                                  │
> │                                                                         │
> │  1. Scale이 1이 아닐 때:                                                 │
> │     - Bevel 같은 모디파이어가 일정하게 안 먹힘 (한쪽만 두껍게 됨).            │
> │     - 애니메이션 시 모델이 이상하게 늘어남 (Squash & Stretch 오류).           │
> │                                                                         │
> │  2. Rotation이 0이 아닐 때:                                              │
> │     - 로컬 축(Local Axis)이 뒤섞임.                                      │
> │     - "앞으로 가!" 했는데 대각선으로 가는 로봇 팔을 보게 됨.                  │
> │                                                                         │
> │  ☞ 결론: "모든 수학적 연산의 기준점을 깨끗하게 청소하는 것!
> ```
