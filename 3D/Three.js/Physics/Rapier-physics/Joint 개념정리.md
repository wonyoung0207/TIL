# Joint 개념정리

---

>

## 정리

- RigidBody 두 개 사이에 “움직임 규칙”을 걸어주는 연결 고리
- 붙이되 아무렇게나 못 움직이게 규칙을 강제로 적용

------

## 사용 이유 

##### 안했을 경우 

```
[ 팔1 ]    [ 팔2 ]
```

- 서로 독립된 물체로 인식되어 충돌하면 튕겨 나감

##### 사용한 경우 

```
[ 팔1 ] --(경첩)-- [ 팔2 ]
```

- 떨어져도 같이 움직이고 특정 축으로만 회전하도록 설정할 수 있음 
- 현실적인 “관절” 느낌으로, **하나의 구조물** 이 됨 

------

## 비유

##### 문

- 문: RigidBody
- 문틀: RigidBody
- 경첩: Joint

- 문은 회전만 가능
- 앞뒤로 날아다니지 않음

### 로봇팔

```
Base ─(joint)─ Arm1 ─(joint)─ Arm2 ─(joint)─ Grip
```

- 이걸 가능하게 만드는 게 **Joint**

------

## 역할

- Joint는 내부적으로 이런 제약을 걸 수 있다. 

1.  위치 제약
   - 두 RigidBody는 이 지점에서 떨어지면 안 된다
2. 회전 제약
   - 이 축 말고는 회전 금지
3. 각도 제한
   - 여기까지만 움직여라
4. 힘 전달
   - 한쪽이 밀리면 다른 쪽도 영향 받음
