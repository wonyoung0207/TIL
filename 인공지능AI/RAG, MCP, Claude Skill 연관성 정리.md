# RAG, MCP, Claude Skill 연관성 정리

---

>

## RAG란?

- 유저 입력(**프롬프트)만으로는 부족**한 경우, 외부에서 관련 정보를 검색(Retrieval)해 **컨텍스트로 주입**한 뒤 그 정보를 바탕으로 **답변(Generation)을 만드는 방식.**

---

## MCP (Model Context Protocol)

###### MCP란?

- LLM이 외부 시스템(파일, DB, Git, SaaS 등)에 표준화된 방식으로 접근하게 해주는 프로토콜.
- RAG가 필요로 하는 “외부 정보 검색/읽기”를 **MCP의 툴 호출(tool invocation)** 로 수행할 수 있게 한다.

###### 요약

- RAG = 정보를 가져오는 개념
- MCP = 정보를 가져오기 위한 표준 통로(도구 세트)

------

## Claude Skill

##### Claude Skill이란?

- LLM이 언제·어떻게 컨텍스트를 가져올지 설계한 “지능형 워크플로우 + 도구 사용 전략”
- 즉, Skill은 단순한 지침이 아니라
  1. 필요 시점에
  2. 어떤 MCP 툴을 호출해
  3. 어떤 정보를 컨텍스트로 넣고
  4. 어떻게 정제·요약할지
  5. 를 **코드 수준의 로직으로 정의한 컨텍스트 엔지니어링 도구**이다.

##### 요약

- **RAG의 검색 전략 + MCP의 외부 접근 + 컨텍스트 최적화 로직** → 이 세 가지가 하나의 “Skill”로 묶여 있는 구조

------

## 전체 관계 요약

- **프롬프트는 유저 입력이고, RAG·MCP·Skill은 컨텍스트를 조립하는 기술/도구**

```
[유저 입력] = 프롬프트

RAG = 외부에서 정보를 검색해 컨텍스트로 주입하는 패턴
MCP = 그 정보를 실제로 가져오기 위한 표준화된 접근 통로
Claude Skill = MCP를 활용해 컨텍스트를 언제·어떻게 주입할지 정의한 워크플로우
```
