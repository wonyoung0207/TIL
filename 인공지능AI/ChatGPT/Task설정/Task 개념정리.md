# ChatGPT Task 개념 정리

## 1. Task란?

> **Task**는 지금 바로 실행되는 요청이 아니라, **미래의 특정 시점 또는 주기적으로 실행되도록 예약된 작업**이다.

- 사용자의 개입 없이 자동 실행
- 시간 기반 / 조건 기반 실행 가능
- 비동기 작업 단위
- 즉, 
  - **Task = 미래에 실행될 AI 작업**

---

## 2. Task가 필요한 이유

- 기존 ChatGPT 대화는 요청 시점에만 동작한다. 
- Task를 사용하면 다음이 가능해진다.

- 특정 시간 알림
- 정기적인 요약·점검
- 조건 만족 시 자동 통지

즉, **기억하고 기다렸다가 행동하는 ChatGPT**를 만들 수 있다.

---

## 3. Schedule이란?

- **Schedule**은 Task가 *언제 실행될지*를 정하는 규칙이다.

> **Task = 할 일**  
> **Schedule = 언제 할지**

---

## 4. Schedule 유형

### 1️⃣ 단발 실행

- 지정된 날짜·시간에 한 번 실행
- 예:
  - 내일 오전 9시에 알림

---

### 2️⃣ 반복 실행

- 매일 / 매주 / 매월 등 주기적 실행
- 예:
  - 매일 아침 일정 요약
  - 매주 월요일 리포트 생성

---

### 3️⃣ 조건 기반 실행

- 일정 주기로 조건을 확인
- 조건 충족 시 Task 실행
- 예:
  - 가격 변동 시 알림
  - 새 공지 등록 시 통지

---

## 5. Task 실행 흐름

1. Task 생성
2. Schedule 등록
3. 대기 상태
4. Schedule 트리거 발생
5. Task 실행
6. 결과 전달

---

## 6. Task 구성 요소

| 구성 요소 | 설명                |
| --------- | ------------------- |
| Title     | Task 식별 이름      |
| Prompt    | 실행 시 수행할 지시 |
| Schedule  | 실행 시점/주기      |
| Condition | 실행 조건 (선택)    |
| Output    | 알림·요약 결과      |

---

## 7. 단순 알림과의 차이

| 구분        | 단순 알림 | Task         |
| ----------- | --------- | ------------ |
| 실행 조건   | 시간만    | 시간 + 조건  |
| 실행 내용   | 고정 문구 | AI 추론·요약 |
| 자동화 수준 | 낮음      | 높음         |

---

## 8. 핵심 요약

- Task는 **스케줄링된 AI 작업**
- Schedule은 Task 실행을 트리거
- 단발 / 반복 / 조건 기반 실행 가능
- Task는 알림을 넘어 **자동화된 AI 행동**을 수행한다
