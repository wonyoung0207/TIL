# RAG 개념정리

>

## 1. 용어 정의

##### 프롬프트 (Prompt)

- 오직 유저 입력만을 의미한다.
- LLM이 질문을 이해하는 *출발점*이다.
- 예) `v3.2.1 버전 변경사항 요약해줘`

> 프롬프트는 유저가 제공하는 입력이며, RAG로 불러온 문서나 스킬이 주입하는 정보는 **프롬프트가 아니라 컨텍스트**이다.

##### 컨텍스트 (Context)

- 모델이 답변을 더 잘하기 위해 **프롬프트에 추가로 제공되는 정보들**
- 외부 문서, 코드, DB 조회 결과 등이 포함된다.
- RAG·MCP·Skill의 목적은 “필요한 컨텍스트만 똑똑하게 제공하는 것”이다.

------

## 2. RAG (Retrieval-Augmented Generation)

##### RAG란?

- 유저 입력(**프롬프트)만으로는 부족**한 경우, 외부에서 관련 정보를 검색(Retrieval)해 **컨텍스트로 주입**한 뒤 그 정보를 바탕으로 **답변(Generation)을 만드는 방식.**

##### 핵심 흐름

1. 유저 프롬프트 확인
2. 외부 지식원(문서/DB/검색엔진 등)에서 관련 자료 검색
3. 검색 결과 중 필요한 부분만 컨텍스트로 주입
4. **프롬프트 + 컨텍스트를 기반으로 답변 생성**

##### 정리

- RAG는 프롬프트를 “확장”하는 것이 아니라, 프롬프트 **옆에 컨텍스트를 붙이는 구조**가 더 정확하다.
- 즉,
  - 프롬프트 = 유저 입력
  - 컨텍스트 = RAG로 가져온 외부 정보
