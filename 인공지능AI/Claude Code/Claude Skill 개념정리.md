# Claude Skill 개념정리

---

>

## Claude Skill이란?

- Claude Skill은 LLM에게 “**필요한 시점에 필요한 정보만 단계적으로 제공**”하는 Context Editing 기반의 도구 세트이다.
- 기존처럼 대량의 문서를 한 번에 프롬프트로 넣는 방식이 아니라, **정말 필요한 정보만 최소한으로 LLM에게 점진적으로 공급하기 때문에 훨씬 적은 토큰으로 높은 효율을 제공한다.**
- 즉,“LLM이 필요한 순간에 필요한 만큼만 정보를 받아 작업하는 최적화된 컨텍스트 관리 도구”

---

## 점진적 공개(Progressive Disclosure) 방식

- Claude Skill은 다음 **3단계 프로세스**로 작동한다.
- 이 점진적 공개 흐름 때문에 Claude가 훨씬 집중력 있게 문제를 해결할 수 있고, 대규모 프로젝트나 방대한 문서도 **오버헤드 없이 처리 가능**해진다.

##### 1. 메타데이터 탐색 (Metadata Discovery)

- LLM은 전체 문서·코드·리소스의 내용 전체를 받지 않고, **구조와 요약 정보(메타데이터)만 먼저 확인**한다.→ 초기 토큰 사용량 최소화

##### 2. 관련 Skill 선택 (Skill Selection)

- LLM이 현재 작업과 가장 관계 있는 Skill을 스스로 판단해 선택한다. → 필요한 부분만 활성화됨

##### 3. 추가 리소스 로드 (On-demand Context Loading)

- 선택된 Skill이 요구될 때에만 해당 Skill이 가진 문서/코드/설정 등 **추가 정보를 단계적으로 LLM에 공급**한다. → 정보 과부하 없이 핵심 내용만 이해 가능

------

## 장점 

- 단순 지침이 아닌 “지능형 에이전트 워크플로우” 이다. 
  - Claude Skill은 단순히 텍스트 설명을 담는 도구가 아니다.
- Skill 내부에는 다음이 포함될 수 있다
  1. 실행 가능한 코드
  2. API 호출 템플릿
  3. MCP tool 연동
  4. 특정 작업 절차(workflow)
  5. 규칙 기반의 지능형 동작 로직
- 이 덕분에 LLM은 Skill을 통해 다음과 같은 동작을 스스로 수행할 수 있다
  1. 적절한 도구를 골라 실행
  2. 필요한 리소스를 호출
  3. 코드를 직접 생성·수정
  4. 외부 시스템과 연동
  5. 상황에 따라 다른 Skill을 재귀적으로 사용

즉,

> **Claude Skill = LLM이 자율적으로 필요한 워크플로우를 구성하고 수행하게 만드는 지능형 에이전트 레이어**
