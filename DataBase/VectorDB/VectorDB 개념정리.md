# Vector DB 개념 정리

---

>

## Vector DB란?

- **Vector DB(벡터 데이터베이스)** 는 텍스트·이미지·로그 같은 데이터를 **의미를 표현한 숫자 벡터(Embedding)** 로 저장하고, **의미적으로 가장 유사한 데이터를 빠르게 검색**하기 위한 데이터베이스이다.

> 텍스트를 그대로 저장하는 것이 아니라
> 텍스트 → 의미 벡터로 변환해 저장한다

------

## Vector DB vs RDB 차이점

| 구분        | RDB (관계형 DB)  | Vector DB         |
| ----------- | ---------------- | ----------------- |
| 저장 기준   | 값, 구조         | 의미(벡터)        |
| 데이터 형태 | Row / Column     | Vector + Metadata |
| 검색 방식   | 정확 매칭        | 유사도 검색       |
| 인덱스      | B-Tree           | ANN(HNSW 등)      |
| 강점        | 정합성, 트랜잭션 | 의미 기반 검색    |
| 사용 목적   | 정형 데이터      | AI / 의미 검색    |

- RDB는 “무엇이 같은가"
- Vector DB는 “무엇이 비슷한가”를 찾는다

------

## 저장 데이터 구조

- Vector DB는 보통 아래 3가지를 함께 저장한다.

```json
{
  "id": "item_001",
  "vector": [0.21, -0.34, 0.88, ...],
  "metadata": {
    "category": "ADAS",
    "module": "ACC",
    "timestamp": "2026-01-05"
  },
  "text": "ACC는 앞차와의 거리를 자동으로 유지하는 기능이다."
}
```

### 구성 요소

- **vector**: 의미를 담은 숫자 (검색 핵심)
- **metadata**: 필터·조건용 구조화 정보
  - Vector DB의 metadata는 자동으로 생기지 않으며,
  - 검색과 필터링을 위해 데이터 관리자가 의도적으로 정제해서 넣는 것이 정석이다.
- **text**: 원문 데이터 (결과 표시용)

---

## 저장 흐름

1. 원본 데이터 수집
   - 로그, 이미지 같은 데이터를 벡터로 저장 
2. metadata 후보 추출
3. 정제 / 표준화
4. embedding 생성
5. Vector DB 저장

------

## Vector DB 데이터 저장 예시

- 자율주행 ACC 이벤트 로그

```json
{
  "vector": [...],
  "metadata": {
    "vehicle_id": "CAR_01",
    "scenario": "ACC_follow",
    "road": "highway"
  },
  "text": "고속도로 주행 중 앞차 급감속으로 ACC가 감속함"
}
```

- 의미 검색 : “고속도로에서 ACC 문제 발생한 사례” → 문장이 달라도 유사한 이벤트 검색 가능

------

## 한 문장 요약

- Vector DB는 데이터를 **의미 벡터**로 저장해  ‘**비슷한 의미의 데이터**’를 빠르게 찾는 AI 시대의 데이터베이스다.
