## 오케스트레이터(Ochestrator)

### 정의

- **오케스트레이터란 컨테이너화 된 애플리케이션의 구성요소들을 배포, 관리 및 확장에 있어 자동화해주는 시스템**을 말한다. 
- 즉, **컨테이너의 관리를 자동화** 해주는 시스템 이라고 생각하시면 될거같습니다. 
  - 자동화란,  다양한 작업과 프로세스를 자동으로 수행하여 애플리케이션을 효율적으로 운영하고 유지하는 것을 의미한다. 
- CNS 의 솔루션으로 사용되는 것이 오케스트레이터 이다. 

### 사용이유

- **애플리케이션의 구성요소들을 효율적으로 조정하고 제어하여 애플리케이션이 원활하게 실행되도록 보장하는 것을 의미.**
- 애플리케이션의 구성요소들은 컨테이너화 되어 있어 관리 시스템인 오케스트레이터가 필요하다. 
  - 이러한 오케스트레이션 시스템은 스토리지 시스템과 상호 작용하기 위해 액세스 인터페이스를 사용한다. 

### 오케스트레이션 vs 오케스트레이터

1. 오케스트레이션
   - 여러 개의 독립적인 작업이 필요한 **복잡한 프로세스를 자동화하고 관리하는 과정**
   - 여러 다양한 작업이나 구성요소를 효율적으로 조정하고 관리하여 원활한 프로세스나 작업을 수행하는 과정을 의미
2. 오케스트레이터
   - **오케스트레이션을 실제로 수행하는 도구나 시스템**
3. 정리
   - **오케스트레이션은 복잡한 프로세스의 조율 및 자동화 작업을 의미하며, 오케스트레이터는 이러한 작업을 수행하는 도구나 시스템이다.** 

### 오케스트레이터 종류 ( PPT page 추가 )

- 오케스트레이터의 종류는 2가지가 있다. 
  1. 컨테이너 오케스트레이터
     - 클라우드 애플리케이션을 컨테이너화 시켜 **배포/관리** 한다. 
     - 여러 개의 컨테이너로 이루어진 애플리케이션을 관리하고, 클러스터 내에서 자동으로 컨테이너들을 배치하고 확장하며, 네트워크와 서비스 디스커버리 등을 처리하는 역할을 한다. 
     - 대표적으로 쿠버네티스가 오케스트레이터이다. 
       - 쿠버네티스, Docker Swarm, Apache Mesos 가 있다. 
       - (그림 삽입 )
  2. **스토리지 오케스트레이터**
     - **데이터와 스토리지 리소스를 관리**하고 자동화 하는데 사용되는 도구이다. 
     - 스토리지 리소스의 프로비저닝, 확장, 관리, 모니터링, 백업 등을 자동화하기 위해 사용
     - 목표
       - 스토리지 리소스를 컨테이너와 애플리케이션에 연결하고 관리함으로써 애플리케이션 배포와 관리를 더욱 간편하게 만드는 것
       - 컨테이너화된 애플리케이션과 함께 작동하여 스토리지 관리를 용이하게 해주고, 이로써 클라우드 네이티브 환경에서의 스토리지 프로비저닝과 관리를 효율적으로 처리할 수 있도록 도와준다. 
     - ROOK,  Ceph , Gluster , Portworx, OpenEBS 가 있다.
     - (그림 삽입)
     - 현재 세미나에서는 스토리지 오케스트레이터에 중점을 두고 보겠다. 

### 오케스트레이션 주요 목표 ( PPT page 추가 )

1. 자동화
   - 애플리케이션 배포, 확장, 관리, 복구 등의 작업을 자동화
2. 가용성 및 확장성
   - 애플리케이션을 필요한 만큼 확장하고 필요 시 장애에 대한 복구를 수행
3. 부하 분산
   - 자원 활용도를 최적화하고 서버의 부하를 분산하여 성능을 개선
4. 서비스 디스커버리 및 로드 밸런싱
   - 클라이언트가 애플리케이션 서비스의 위치를 찾고 요청을 분산시키는 작업을 수행
5. 설정관리 
   - 서로 다른 환경에서도 일관된 설정 관리를 제공하여 배포의 일관성을 유지

- CNS 와 오케스트레이터의 연관점
  - 클라우드 환경에서 애플리케이션의 배포, 관리, 스토리지 관리 등을 효율적으로 지원하는데 사용되며, 함께 연계되어 클라우드 기반의 애플리케이션을 운영하기에 유용한 요소들이다.
